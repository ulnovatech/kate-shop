# Kate Admin PWA (C9)

Kate Admin is an **online-only staff client**. It has its own install identity, separate from the shop PWA.

## Shop vs Admin

|                    | Kate Shop                      | Kate Admin                                                                      |
| ------------------ | ------------------------------ | ------------------------------------------------------------------------------- |
| **Origin**         | `shop.yourdomain.com`          | `admin.yourdomain.com`                                                          |
| **Manifest**       | `/manifest.webmanifest`        | `/manifest.webmanifest` (admin app) or `/admin-manifest.webmanifest` (monolith) |
| **Icon**           | `/pwa-icon.svg`                | `/admin-icon.svg`                                                               |
| **Name**           | Shop name from settings        | **Kate Admin**                                                                  |
| **Service worker** | Workbox (offline browse)       | **None** — shop SW evicted on admin origin                                      |
| **Install prompt** | Storefront banner (C4)         | Never shown                                                                     |
| **Offline**        | Partial browse + cached images | Banner only — all staff actions need network                                    |

## Staff manifest

Generated by `buildAdminWebManifest()` in [`src/lib/pwa.ts`](../src/lib/pwa.ts):

- `display: standalone`
- `theme_color` / `background_color`: `#064e3b` (emerald)
- `id`: `/` on `apps/admin`, `/admin` on monolith — prevents conflating with shop install
- Icons: [`apps/admin/public/admin-icon.svg`](../apps/admin/public/admin-icon.svg)

Add to home screen on `admin.yourdomain.com` → installs **Kate Admin**, not the shop.

## Service worker policy

**`apps/admin`** and **`admin.*` subdomain** (`kate-admin` Worker):

- No `vite-plugin-pwa` / Workbox in the admin build
- `StaffPwaPolicy` unregisters any shop service workers and clears Workbox caches on load
- Staff APK (Capacitor) loads the remote admin URL — same policy applies in the WebView

**Monolith** (`npm run dev:monolith`) on the shop host:

- Shop Workbox remains for storefront offline
- Admin JS chunks are excluded from precache (`globIgnores: admin-routes-*, admin-ui-*`)
- `/admin/*` paths stay online-only via `OFFLINE_BLOCKED_PREFIXES`

## Offline UX

`StaffOfflineBanner` shows when `navigator.onLine` is false:

> You are offline. Kate Admin needs an internet connection for orders, inventory, and staff actions.

Dev server hint on admin login uses `npm run dev:admin` (see `staffDevServerUnreachableMessage()`).
